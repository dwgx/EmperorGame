# EmperorGame Architecture & Runtime Notes

## Overview
- Backend: Java (HTTP + WebSocket), MySQL (`users`, `registration_log`, `login_log`, `session_tokens`).
- Ports: HTTP `13338`, WS `13337` (override via `HTTP_PORT` / `WS_PORT`).
- Frontend: prebuilt static pages (`src/main/web`), can be served by the embedded HTTP server or Nginx.
- Reverse proxy (recommended): `/` static, `/api/` → HTTP port, `/ws/` → WS port; on HTTPS use WSS via `/ws/`.

## Runtime Flow
1) Login/Register → `/api/login` or `/api/register` → returns `accountId` + `token`.
2) Frontend caches `accountId`/`token` + last good API base; WS connects and sends `set_profile`.
3) Matching: `match_online` queues; pairing creates `GameRoom`.
4) Play: `play_online` locks cards; when both ready, resolve outcome → broadcast scores/cards/remaining.
5) End: room scores finalized, RoomManager cleans up.

## Backend Core
- `EmperorServer` (WebSocket): `onOpen`, `onMessage` dispatch; `handleSetProfile` binds nickname/account; simple rate-limit.
- `RoomManager`: queue + room lifecycle; `match`, `leaveRoom/retireRoom`.
- `GameRoom`: holds `Deck`, scores, rounds, public flag; `play()` validates, locks, judges, broadcasts.
- `AuthService`: `register/login/validate` → `{ok,message,accountId,token}`; token HMAC-SHA256 (24h) stored in `session_tokens`.
- HTTP: embedded HttpServer, `/api/*` POST only, CORS echoes Origin, serves classpath/static resources.

## Frontend Core (Vue)
- `App.vue`: state for prefs/session/ui; API calls same-origin `/api`; WS connects to `wss://<current-host>/ws/` (no hardcoded port), auto `set_profile`.
- `LoginPage.vue`: forms for `/api/login` `/api/register`, caches `accountId`/`token`; no auto `/api/validate` dependency.
- `RulesPage.vue`: rules + single MP4 demo, minimal controls.

## Game Rules (playful)
- Chain: Emperor > Citizen > Slave > Emperor; Traitor only beats Emperor, loses otherwise; Madman = draw + random event.
- End: Emperor loses a hand = lose match; Slave beats Emperor = win match; deck empty = end.
- Madman random events (on play): swap one random card each; or both discard one random card; or redo this hand (Madman still consumed).
- Public round: Traitor/Madman trigger next round to reveal locked cards immediately.
- Deck modes: Standard 1E/4C/1S/1T/1M; Random 1E/4–7C/1–2S/1–2T/1–2M (toggle `/set randomcard on|off` or `DECK_MODE=random|standard`).

## Build & Run
- Package: `mvn clean package` → `target/EmperorGame-1.0-SNAPSHOT-shaded.jar` (exec jar with manifest).
- Run: `java -jar target/EmperorGame-1.0-SNAPSHOT-shaded.jar`
- Custom ports: `WS_PORT=80 HTTP_PORT=81 java -jar ...`
- DB: set `DB_URL/DB_USER/DB_PASS`; tables auto-created.
- Frontend rebuild (after source changes): `cd src/main/frontend && npm install && npm run build`, then deploy `dist` into `src/main/web` or your web root.

## Nginx (japanese.icu sample)
```nginx
server {
    listen 80;
    server_name japanese.icu;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name japanese.icu;

    root /www/wwwroot/japanese.icu;
    index index.html;
    try_files $uri $uri/ /index.html;

    location /api/ {
        proxy_pass http://127.0.0.1:13338;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /ws/ {
        proxy_pass http://127.0.0.1:13337/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Troubleshooting
- API fails: ensure proxy `/api/` → 13338; if backend down, frontend will now treat non-2xx or `ok!==true` as error.
- WS fails: ensure `/ws/` proxy to 13337 with Upgrade headers; on HTTPS the frontend uses `wss://<host>/ws/`.
- “No main manifest” jar: use the `*-shaded.jar` generated by `mvn clean package` (shade plugin now attaches executable jar).
